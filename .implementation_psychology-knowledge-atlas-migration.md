## 0. 已確認的目標與原則

- 使用者目標：蒐集心理學所有學派與相關內容，建立超越一般教科書深度的內容、標籤與索引，未來與 `religions-history` 對接。
- 大量蒐集本身不是問題；錯誤是把 AI 生成文章當成蒐集成果。新版蒐集單位必須是真實來源、實體、證據與關係。
- LLM 不作知識來源。LLM 可協助搜尋詞、在已提供來源內提出候選摘取、整理核准 claims、找缺口；不能自行宣告內容已證實。
- 研究材料不必具有公開再散布授權才能閱讀、分析與重新組織；公開 repo 不保存未授權全文或大量原文。
- 不破解 DRM、不繞過付費牆、不共用帳號、不尋找盜版下載站、不建立未授權全文資料庫。
- 舊 `psychology-schools` 不刪除、不改寫 Git 歷史；它保留為明確警告的 legacy archive。
- 新 repo 使用全新 Git history；舊文章不得搬成新版正式知識。

## 4. 新 repo 的核心設計

### 4.1 資料層

```text
psychology-knowledge-atlas/
├── README.md / PLAN.md / CLAUDE.md / HANDOFF.md / MAP.md
├── schemas/             # entity/source/evidence/claim/relation/vocabulary/coverage
├── catalog/             # 心理學實體宇宙與收錄裁決
├── library/             # 書目、版本、取得狀態；不放不可公開全文
├── knowledge/           # claims、evidence、relations、controversies
├── vocabularies/        # 多軸受控詞彙
├── crosswalks/          # D1–D13 與 religions-history 合約
├── views/               # 可重生文章、時間線、比較、索引
├── tools/
└── tests/
```

canonical storage 優先採每實體／每來源獨立檔，避免多 agent 寫同一巨大 JSONL。SQLite、索引 JSON、Markdown 與網站資料均為可重生產物。

`.gitignore` 至少排除：

```gitignore
.research-cache/
.private-sources/
*.pdf
*.epub
*.mobi
```

### 4.2 實體類型

- `tradition`、`school`、`paradigm`、`theory`、`model`
- `therapy`、`subfield`、`method`、`construct`
- `person`、`work`、`institution`、`event`
- `finding`、`controversy`

現有 48 項只能作候選種子，必須重新判型、去重與建立 alias／parent／branch／influenced／opposed 關係。

### 4.3 操作性完整

「所有學派」定義為：指定多套權威參考系後，對其中所有候選達成 100% 收錄／合併／排除／待裁決紀錄，並保留持續 discovery queue；不宣稱存在客觀封閉全集。

參考系至少涵蓋：心理學通史、學派與理論 handbook、心理治療 handbook、各次領域 handbook、非西方／本土／跨文化心理學史、專業學會與學術百科／辭典／書目資料庫。

### 4.4 來源與研究輸入

來源取得狀態與證據品質分開：

- `open_fulltext`
- `public_domain_fulltext`
- `repository_manuscript`
- `preprint`
- `publicly_readable_license_unclear`
- `privately_observed_unredistributable`
- `abstract_only`
- `metadata_only`
- `paywalled_unread`
- `unavailable`

可閱讀且身份／版本可驗證的材料可作研究輸入；是否可再發布只控制全文保存，不自動否定知識內容。搜尋 snippet 與 metadata 不得冒充讀過全文。

公開 repo 只保存正確書目、版本、locator、必要短摘錄、自己的證據摘要、原子 claims、relations、索引與重組文章。

### 4.5 內容模組

每個主題依證據選擇模組，不強迫填滿：定位與邊界、歷史、譜系、人物、原典、基本假設、概念、命題、機制、研究方法、測量、治療技術、適用族群、實證、反例與再現、歷史／科學／文化批評、教科書扭曲、鄰近學派比較、當代遺產、未解問題、D1–D13、宗教／靈性接口與適用邊界。

「超越教科書」以原典與學術史、內部分歧、版本演進、方法細節、證據層級、反例、非西方觀點、跨學派比較及透明缺口衡量，不以字數衡量。

### 4.6 多軸標籤

至少分開：實體類型、歷史時期、地區、文化範圍、思想譜系、分析層次、研究對象、方法、應用、族群、證據類型、證據狀態、心理構念、D1–D13、religion-interface relation。自由 keywords 只供搜尋。

### 4.7 religions-history 對接

- 保留 D1–D13 作穩定共同軸。
- 不再只做「經文 psych_tag → D 碼 → 拉出所有學派」的粗 join。
- 新增有 claim/evidence 的 relation：`explains`、`contextualizes`、`compares_with`、`contrasts_with`、`empirical_research_on`、`clinical_application`、`appropriation_risk`、`not_applicable`。
- 第一階段不修改 religions-history 的經文與翻譯；新版 cross-project contract 通過後才切換 consumer。

## 6. 必要測試

- schema：空、壞 JSON、重複 ID、未知欄位、舊到新 migration。
- referential integrity：alias、relation、tag、claim、evidence 無 orphan。
- 來源：API 空回應、429、截斷、malformed response 不產生假來源。
- 證據：metadata-only／snippet 不得支持全文 claim；locator 必須存在；複合 claim 被拒。
- 寫作：生成文章新增無 claim 的具體事實時 build 失敗。
- regression：舊庫已知假人物、假書目、錯年、錯誤歸屬必須被攔截。
- boundary：空、超大、中斷、半完成輸入。
- concurrency：兩程序寫不同實體不衝突；同一實體正確加鎖。
- reproducibility：刪除 generated views 後能由 canonical data 完整重生。
- cross-project：D1–D13、細標籤與心理實體 join 無 orphan，語意版本一致。
- Git safety：PDF／private cache 不會被追蹤；舊 Git 歷史未改寫；Axline 修改仍存在。
