## 步驟 4｜預期風險

### 4a. 執行時風險

1. **學術史與理論歸屬的混淆**
   - 風險位置：第 1、2、3 段。
   - 原因：正向心理學在 1990 年代末以新運動形式成形，但其思想前史涉及 William James、人本心理學、Ryff 的心理幸福感、Diener 的幸福感研究等；若把所有前身直接說成「正向心理學創始者」，會造成歷史過度簡化。
   - 具體預防：分開「前史來源」「正式運動的制度化」「後續理論發展」，並把 Seligman、Csikszentmihalyi 的貢獻限定在可合理歸屬的範圍。

2. **把正向介入或幸福感研究的效果說得過強**
   - 風險位置：第 5、6、8 段。
   - 原因：正向心理學文獻常出現小型研究、短期自陳結果、出版偏誤與重複研究差異；若把「平均有幫助」寫成「普遍有效」或「能治療心理疾病」，會超出實證支持。
   - 具體預防：避免無法核實的單一效果數字；區分相關、短期介入效果、臨床療效與長期功能改善；將證據分歧明列於第 8 段。

3. **繁體中文與固定格式的機械檢查失敗**
   - 風險位置：全文輸出與驗證步驟。
   - 原因：模型可能漏掉標題、使用近義標題、混入簡體字，或在正文前加入自我說明；專案 verifier 也可能對特定繁體用字產生粗略誤報。
   - 具體預防：寫入後重新讀取全文，逐一檢查八個標題、第一字元與簡體字粗篩結果；任何失敗先修檔再交付。

### 4b. 結構性風險

- **並發**：不適用。本次不新增 lock、共用狀態或並行寫入；只在現有工作樹中修改一份內容檔。
- **邊界**：適用於內容完整性而非程式輸入。空白檔、截斷檔、少於八段的檔案會在讀取與 verifier 階段被攔截。
- **持久化**：不適用。不改 state schema、資料庫或序列化格式。
- **外部輸入驗證**：部分適用。正文涉及學術史與研究結論，但不接收即時 API 回應或使用者輸入；會以來源等級與確定性標記限制未核實宣稱。
- **命令注入**：不適用。不組合 shell、SSH、SQL 或外部命令中的使用者輸入。
- **跨狀態一致性**：不適用。不新增或修改任何狀態欄位、排程狀態或索引 schema。

3. **針對格式或文字驗證失敗**

     - 若有前言，刪除前言使檔案第一字元直接為 `#`。
