## 步驟 3｜執行路徑

1. **確認目前草稿狀態**
   - 工具：讀取 `schools/health-psychology/synthesis.md`。
   - 前置條件：檔案目前存在但內容為空。
   - 若發現已有實質內容，先保留並重新評估是否需要避免覆蓋；不會直接抹除未確認的人工成果。

2. **整理健康心理學的研究範圍與歷史脈絡**
   - 以健康心理學正式形成、醫療模式轉向、健康行為與疾病管理等軸線組織內容。
   - 區分健康心理學作為跨學科領域與單一治療學派的差異，避免把心理生物社會模式、臨床心理學或行為醫學直接等同於健康心理學。
   - 核對其與生理醫學、行為醫學、社會心理學、認知行為取向及公共衛生的承接關係。

3. **建立人物與理論來源清單**
   - 規劃納入可查證的代表人物與理論，如 Joseph Matarazzo、George Engel、Irwin Rosenstock、Howard Leventhal、Albert Bandura、Icek Ajzen、James Prochaska、Carlo DiClemente、Richard Lazarus、Susan Folkman 等。
   - 每位人物只納入與健康心理學直接相關且可確認的貢獻。
   - 生卒年、正式定義、理論提出年代與數字性宣稱，若不能達到高可信度，降為 🟡／🟠／🔴 或寫「資料待補」。

4. **撰寫八段正文**
   - 工具：使用 `Edit` 修改既有 `synthesis.md`；若檔案確認為空白，可使用 `Write` 完整建立正文。
   - 內容包括：
     - 健康與疾病的連續性及多層次決定因素。
     - 健康行為、疾病表徵、壓力與因應、自我效能、健康信念、計畫行為及行為改變階段等核心理論。
     - 健康促進、疾病預防、醫療遵從、慢性病自我管理、病人—醫療者溝通與健康不平等等方法與應用。
     - 學界對個人責任化、測量偏差、文化外推、因果推論、介入效果與健康不平等的批評。
     - 主要回應 D9，並說明如何延伸至 D13。
   - 嚴格避免把理論條列當作完整展開；每個核心理論都需說明問題意識、運作機制、預測內容與實務意義。
   - 只輸出 Markdown 正文，不在檔案開頭加入「已完成」或其他自我說明。

5. **進行內容與格式驗收**
   - 工具：`Read`、`Grep`，必要時執行 `PYTHONIOENCODING=utf-8 python tools/verify.py`。
   - 核對八段標題、第一字元、繁體中文、來源標記、確定性標記、D9／D13 對接、書目版權標記與禁用圖表內容。
   - 對照 SOP 附錄 A；本篇主要不直接涉及該清單中的八項典型扭曲，但會在 §8 明確說明檢查結果。
   - 若檢查發現遺漏，直接修正後重新驗收。

6. **完成原始要求對齊檢查**
   - 逐條確認：
     - 第一字元是 `#`。
     - 八段標題原樣保留。
     - 全程繁體中文。
     - 沒有前言、結尾語、自我說明。
     - 沒有圖表或製圖內容。
     - 第 3 段是完整理論展開。
     - 第 4 段含書名、作者、年代及 PD／©。
     - 第 7 段回應 D9／D13。
     - 第 8 段誠實列出缺口與爭議。

## 步驟 4｜預期風險

### 4a. 執行時風險

1. **狀態假設風險**
   - `synthesis.md` 雖然目前讀取結果顯示為空白，但它位於未追蹤目錄中，可能是其他流程預備中的檔案。
   - 若直接覆寫未重新確認，可能損失尚未提交的內容。

2. **研究事實與來源風險**
   - 健康心理學是跨學科領域，正式形成年代、代表人物範圍及「第一個」之類的歷史說法容易把行為醫學、醫學心理學與健康心理學混為一談。
   - 若只依單一教科書或百科，可能產生錯誤歸屬、過度簡化或捏造書目。

3. **範圍與分類風險**
   - 健康心理學並非單一治療技術，若沿用其他治療學派的寫法，可能過度臨床化，忽略公共衛生、健康政策、醫療系統與健康不平等。
   - 若只集中在個人健康行為，可能把疾病責任不當歸因於個人，漏掉社會結構與醫療可近性的影響。

### 4b. 結構性風險

- **並發**：不適用。本次不新增 lock、共用狀態或跨進程寫入；但會先確認目前檔案狀態，避免與引擎同時產生內容。
- **邊界**：適用於內容完整性。空白檔、截斷正文、缺少 §3 或 §8、非 UTF-8 字元、標題被模型改寫，都必須在驗收時捕捉。
- **持久化**：不適用於 schema migration。本次只寫 Markdown，不改 JSON schema、資料庫或狀態格式。
- **外部輸入驗證**：適用。研究資料、人物年代、書目及理論名稱屬外部事實輸入；無法確認的內容必須降級或列為資料待補，不直接收為確定事實。
- **命令注入**：不適用。本次不組合 shell command、SSH、SQL 或外部 API 請求。
- **跨狀態一致性**：不適用於程式狀態；但需確保正文對接的 D9／D13 與現有 `crosswalk-13-domains.md` 一致。

## 步驟 5｜風險預案

1. **針對草稿狀態風險**
   - 提前偵測：寫入前再次讀取 `synthesis.md`，並檢查 git status，確認檔案是否仍為空白以及是否有其他未提交內容。
   - 發生後處理：若出現實質既有內容，停止覆寫，保留原稿並請確認；若仍為空白，才寫入全文。

2. **針對研究事實與來源風險**
   - 提前偵測：先對照 SOP 的三角驗證要求，將正式定義、關鍵年代、人物歸屬與書目分開核對；不使用無法確認的精確數字。
   - 發生後處理：刪除查不到的具體宣稱，或改為 🟡／🟠／🔴 並在 §8 說明待查項目；不以假引用補洞。

3. **針對範圍與分類風險**
   - 提前偵測：撰寫前檢查是否同時涵蓋個人、醫療互動、社會環境與公共衛生層次，並檢查是否明確區分健康心理學與心理生物社會模式。
   - 發生後處理：在 §3、§5 或 §6 補回被遺漏的層次；在 §8 誠實標示尚未充分處理的文化、結構或政策議題，不擴大成未要求的專題。

4. **針對正文結構風險**
   - 提前偵測：寫入後用 Grep 檢查八個精確標題、開頭第一字元、禁用詞及是否存在 `## 步驟`、`已寫入` 等計畫或自我說明殘留。
   - 發生後處理：直接修正正文並重新執行結構檢查；若正文被截斷，依段落重新補齊，不將截斷版本視為完成。

## 步驟 6｜整體檢視

### 1. 有更簡單的路嗎？

有。最簡單且符合範圍的路徑是只修改目前的 `schools/health-psychology/synthesis.md`，不建立新腳本、不改 schema、不修改分類與交接文件。因為本次不是結構性改動，也沒有必要改動引擎或新增自動化工具。

### 2. 範圍有沒有被放大？

沒有。計畫只處理健康心理學一篇綜述與必要的格式／事實驗收；不會順手校核其他草稿、不會改跨學派索引，也不會把健康心理學擴寫成一般醫學史或公共衛生史。

### 3. 驗收標準是什麼？

- 讀取 `schools/health-psychology/synthesis.md`，確認檔案非空白。
- 確認第一個字元為 `#`，且首行為 `## 1. 定位與歷史脈絡`。
- 以 `Grep` 或逐行讀取確認八個標題全部存在、順序正確且沒有被合併。
- 確認正文沒有前言、自我說明、結尾語、圖表、製圖或「如圖」文字。
- 確認第 3 段不是只列定義，而是展開健康心理學的核心理論與機制。
- 確認第 4 段每筆書目含書名、作者、年代及 PD／© 標記。
- 確認第 7 段明確標記並解釋 D9、D13。
- 確認第 8 段列出實際未查證項目、學界爭議、文化與因果限制，且沒有以「我不確定」取代資料狀態。
- 執行 `tools/verify.py` 的可用結構檢查，並以 `Read` 重新確認實際檔案內容。
- 本次 4b 的外部輸入與邊界風險，驗收以來源／確定性標記、八段完整性、空白／截斷檢查與 UTF-8 讀取為對應測試；並發、schema migration、命令注入均明確標示不適用。
