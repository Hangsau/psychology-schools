# AI 證據保證路線圖（P6–P8）

> 目標：不依賴逐篇人工審稿，將內容庫從「AI 產文後 AI 校核」升級為可重跑、可稽核、可量化的證據系統。這不是宣稱 AI 不會錯；而是讓每個錯誤有更高機率被來源、獨立模型與規則攔下，且未被證實的內容不能偽裝成已證實。

## 1. 原則與誠實邊界

- AI 可在**覆蓋率、一致性、重複查驗、規則執行**上優於單一人類審閱。
- AI 不能在沒有可靠來源、來源本身錯誤，或所有模型共用同一錯誤前提時，保證真實。
- 因此品質定義為：每項重要主張都有可重現證據、獨立交叉驗證和明確狀態；不是模型說「我已查過」。
- 臨床療效、診斷、風險與治療建議採最高門檻；本庫以學術綜述為限，不輸出個別醫療建議。

## 2. 新資料模型：主張級而非篇章級

每個 `schools/<slug>/` 新增 `claims.jsonl`（一行一筆主張）。範圍先限高風險主張：人物生卒／事件與出版年代、書目、精確數字、因果與療效、最早／唯一／首次等絕對語句。

每筆至少含：

```json
{"id":"cbt-001","claim":"Beck 於 1976 年出版…","type":"bibliographic","risk":"high","source_ids":["doi:…","isbn:…"],"source_independence":"independent","evidence_quote":"<=25 字定位摘錄","verdict":"supported","confidence":"high","checked_at":"YYYY-MM-DD","checker":"retrieval+verifier-v1","text_anchor":"synthesis.md#4"}
```

`source_ids` 必須是 DOI、ISBN、PubMed、出版社、學會、政府或大學館藏等可重新取得的永久識別／網址；`A/B/C/D` 僅可作為分類，不可替代來源。

## 3. 狀態機（禁止以 `reviewed` 概括一切）

每個主張只有下列狀態：

`unverified` → `retrieved` → `corroborated` 或 `disputed`／`insufficient`。

- `retrieved`：來源可取得，且可定位到支撐材料。
- `corroborated`：兩個真正獨立來源支持；療效主張至少一個為系統性回顧、臨床指引或同等級證據。
- `disputed`：可信來源結論不同，正文必須並列，不可選一說成事實。
- `insufficient`：無足夠證據；正文須刪除、改為待證問題，或降為明確低信心陳述。

`meta.json` 未來以 `evidence_state` 取代含混的整篇 `review_state`：`structure_checked`、`bibliography_checked`、`high_risk_claims_covered`、`evidence_release`。公開頁面只顯示已達門檻的版本。

## 4. P6：建立可重現證據管線

1. 訂定白名單來源層級：原典／出版社、DOI 與 PubMed、學會與政府指引、系統性回顧、學術百科；一般網站只能用於找線索。
2. 建置檢索器：每筆主張先取來源與書目中繼資料，再將來源快照、取得日期與授權限制存成證據索引；不得把模型記憶當來源。
3. 建置宣稱抽取器：從正文標記高風險句，要求每句連到 `claim_id`；新增未連結高風險句即 CI 失敗。
4. 建置確定性驗證器：驗證 DOI/ISBN 格式、URL 可達性、書名作者年代與來源 metadata 一致、兩個來源是否實際獨立、引文位置是否存在。
5. 用 3 篇代表性文章試跑（CBT、精神分析、本土心理學），先校準 schema 和失敗類型，再擴展全庫。

## 5. P7：獨立 AI 驗證，而非同模型自評

每項主張採角色隔離，並保留全部輸入輸出與版本：

1. **寫作者**：只能依已核證據起草；無來源不得補寫。
2. **檢索器**：只回傳來源與精確定位，不寫結論。
3. **驗證器 A**：依來源判斷「是否支撐該主張」。
4. **驗證器 B**：使用不同模型／不同 prompt／不同時間窗，盲測同一主張。
5. **裁決規則**：A/B 與來源一致才升 `corroborated`；不一致一律 `disputed` 或 `insufficient`，不以多數投票硬判真。

模型獨立不只換名稱：必須隔離 prompt、不得把寫作者答案傳給驗證器、優先使用不同來源集合。這可降低「同一幻覺被多次認同」的相關錯誤。

## 6. P8：自動發布閘門與持續品質監測

- CI 分三層：結構檢查、證據完整性檢查、來源再驗證；現有 `verify.py` 只屬第一層。
- 發布門檻：所有高風險主張 100% 有 `claim_id`；至少 95% 為 `corroborated` 或 `disputed`；`insufficient` 不得使用肯定事實語氣。
- 每次內容變更都重新抽取主張、重驗受影響來源；來源失效、撤稿或指引更新時自動降級並重建頁面。
- 建立隱藏式測試集：已知教科書扭曲、曾在本庫發現的假書目與錯年、相近概念張冠李戴。任何版本皆須盲測，不將答案寫進生成 prompt。
- 儀表板公開每篇：高風險主張數、已交叉驗證比例、爭議數、未證數、最近查驗日及證據版本；不再用單一綠色標記代表全部可信。

## 7. 執行順序與停止條件

先完成 P5 的內容補全並凍結正文，再做 P6 的三篇試點。試點必須量測：來源可取得率、主張覆蓋率、驗證器分歧率、已知錯誤攔截率與每篇成本。只有當這些指標達標，才批次擴展到 48 篇。

若無法取得兩個獨立可查來源，正確處置是保留不確定性，不是提高模型次數或把文字寫得更肯定。
